# yourConnector 文档导航

## 1. 文档系统原则

1. 单版本原则：仓库只维护当前有效文档，不保留 `v1/v2` 并行版本。
2. 代码优先原则：先核对代码事实，再更新文档，不以旧文档互抄。
3. 职责分离原则：每篇文档只承载一个主要职责，避免重复描述。
4. 中文优先原则：正文使用中文，协议名/接口名/工具名保留英文原词。

## 5. 维护约束

1. 新增或删除文档必须同步更新本导航与 `scripts/check-doc-consistency.sh`。
2. 新增或重命名关键源码文件必须同步更新 `docs/代码事实总索引.md`。
3. 协议事件改动必须同步更新 `docs/API与事件协议.md`。
4. 配对链路改动必须同步更新 `docs/配对与宿主机接入/`。
5. 运维页交互或状态机改动必须同步更新 `docs/运维与工具管理/`。
6. 聊天/报告链路改动必须同步更新 `docs/聊天与报告/`。
7. 详情 schema 或采集策略改动必须同步更新 `docs/工具详情与数据采集/`。
