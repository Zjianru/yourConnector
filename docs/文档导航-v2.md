## 2. 阅读顺序（推荐）

1. 先读 `README.md` 了解结构、命令与门禁入口。
2. 再读治理规范：`docs/代码治理与注释规范-v1.md` 与 `docs/质量门禁与检查规范-v1.md`。
3. 阅读 `docs/系统日志与归档-v1.md` 了解系统日志落盘与归档规则。
4. 阅读 `docs/分发安装与卸载-v1.md` 了解 Linux 节点安装、证书与卸载策略。
5. 再读 `docs/配对与宿主机接入` 目录理解配对与安全主链路。
6. 继续读 `docs/工具接入核心组件-v1.md` 理解工具详情采集框架。
7. 阅读 `docs/跨宿主联调测试-v1.md` 执行 A/B/模拟器联调。
8. 最后执行 `docs/已完成功能验收-v1.md`，并结合 `docs/里程碑与待办-v1.md` 确认状态。

## 3. 维护约束

1. 历史草案文档已删除，不再作为实现依据。
2. 新增文档必须纳入本导航，否则视为未接入。
3. 协议变更必须同步更新 `docs/配对与宿主机接入/02-协议与安全方案-v1.md`。
4. 验收口径只能在 `docs/已完成功能验收-v1.md` 维护。
